# Purposeful Animation

The goal is not to make everything move. It is to make the right things move at the right time.

Every motion should serve one of three functions:

- **Orient** the user — show where something came from or where it went
- **Give feedback** — confirm a press, a copy, a successful submission
- **Show relationships** — reveal that a card expands into a detail view, or that a list item is part of a group

If an animation does none of these three things, remove it.

A static landing page works. An animated one converts better. But motion must be purposeful — animation that distracts or delays hurts more than it helps.

## The Golden Rule

> "The best animation is that which goes unnoticed."

If users comment "nice animation!" on every interaction, it's probably too prominent for production. (Exception: kids apps and playful contexts where delight IS the goal.)
